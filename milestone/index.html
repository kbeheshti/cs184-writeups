Overview

For our project we will be presenting a 2D smoke simulation simulated by the Navier-Stokes equations. We will use several shaders (i.e. buoyancy, pressure, forces applied, boundary intersections) to efficiently simulate the effects of a 2D skybox-bounded smoke apparatus, with support for colors as well. The user will be able to adjust certain parameters as they see fit, such as modifying the curl fragments, the dissipation rate, or dissipation of forces such as velocity.

Current Progress

So far we have implemented the user interface responsible for interacting with the skybox environment. We are using a fixed orthographic camera to simulate a skybox environment in which the user will use their mouse to click and drag to simulate the smoke. We have also implemented a frame buffer that will be used to handle reading/writing to a WebGLRenderTarget. Since we cannot read and write to a texture concurrently, the buffer is responsible for keeping track of the state of a property through two textures, one for reading from and one to write to, which we then repurpose as changes are written, such that one texture is passed through a shader (write) and then rendered onto the other (read) which is then rendered onto the scene.

Shaders are stored in ./shader and asynchronously loaded in the js program. Mouse-to-pointer operations have also been implemented but are not yet in use. We tested applying a shader by turning the viewport green using a black-to-green animation gradient. We have also tested diffusion separately which we now need to translate into a shader.

Plan

Now that the framework has been laid out, the next step is to write and implement the shaders. There are several shaders we are taking into consideration for implementation:

  • Buoyancy (may be set by default or modified via temperature setting if added)*
  • Vorticity (tail curling at smoke ends)
  • Diffusion (mostly done)
  • External forces (static values assigned on mouse click for density and velocity)
  • Boundary conditions (interacting with skybox borders)
  • Advection (dissipation of density, velocity, and temp)

*Temperature may be added as a modifiable field, however this may complicate other simulations depending on the range of temperatures supported. For now, the smoke temperature will be assigned a constant value and this field may be added for the user if time permits.
